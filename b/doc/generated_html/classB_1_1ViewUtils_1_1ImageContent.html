Detailed Description

Helper class for custom views that display image content.

This is a little wrapper class around the HIViewImageContentInfo structure, which is used by the standard Toolbox controls to get/set image content. It allows implementers of custom views to add the ability to draw image content – icons, PICTs, CGImages – without too much fuss.

The idea is that you add to your custom view a member variable of type Viewutils::ImageContent, and that within the your SetData() function, you assign the incoming HIViewImageContentInfo to the member variable. Likewise, within Draw(), you call one of ImageContent's drawing functions, depending on whether you draw to a GrafPort or a CGContextRef.

Note
Not all content types can currently be drawn to both a GrafPort and a CGContextRef. You can call CanDrawIntoGrafPort() or CanDrawIntoContext() to validate the content type.
The CGImageRef content type is only supported when building on 10.3 or later.
