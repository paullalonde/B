Wrapper class around IconRefs.

The class properly maintains reference counts on the underlying IconRef.

Constructor & Destructor Documentation

B::Icon::Icon ( )

Default constructor. The icon is set to NULL.

An Icon initialised in this way isn't really useable, since there's no underlying IconRef. It's allowed mainly to permit instances of Icon to be placed in containers, and other situations which require a default constructor.

Draws the icon upside down in the given CoreGraphics context.

This function draws the icon using the current transformation matrix of inContext. This usually means that the icon will draw upside down, because CoreGraphics' Y-axis goes up instead of down like Quickdraw.
